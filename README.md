# Aravt · Задачи — Supabase + Realtime + голос через Claude

Таск-менеджер, который вы наполняете **голосом**: диктуете задачу в Claude.ai на iPhone →
Claude через MCP-коннектор пишет её в базу Supabase → сайт на Vercel **мгновенно** показывает
обновление без перезагрузки.

```
Голос в Claude.ai  →  Supabase MCP пишет в БД  →  Supabase Realtime  →  Сайт на Vercel
```

## Что в папке

```
index.html              ← точка входа Vite
vite.config.js          ← сборка
tailwind.config.js      ← палитра и шрифты
postcss.config.js
package.json            ← зависимости
.env.example            ← шаблон ключей (скопировать в .env)
supabase_setup.sql      ← SQL: таблица + доступы + Realtime + 13 задач
src/
  main.jsx              ← запуск React
  App.jsx               ← всё приложение (с комментариями)
  supabaseClient.js     ← подключение к Supabase
  index.css             ← стили Tailwind
preview.html            ← статичный показ дизайна (демо-данные, без базы) — можно открыть двойным кликом
```

> Хотите просто посмотреть, как выглядит интерфейс, прямо сейчас — откройте **preview.html**
> двойным кликом. Это копия дизайна на тестовых данных, база не нужна.

## Шаг 1. Создать проект в Supabase и запустить SQL

1. Зайдите на **supabase.com** → **New project**. Придумайте имя, пароль базы, регион
   (ближе к вам). Подождите ~2 минуты, пока проект создаётся.
2. Слева откройте **SQL Editor** → **New query**.
3. Откройте файл **`supabase_setup.sql`**, скопируйте его целиком, вставьте и нажмите **RUN**.
4. Проверьте: **Table Editor → tasks** — там должны появиться 13 задач.

Этот скрипт сразу: создаёт таблицу, включает Realtime, настраивает права и загружает задачи.

## Шаг 2. Взять Supabase URL и anon-ключ

1. **Project Settings** (шестерёнка) → **API** (или **Data API**).
2. Скопируйте два значения:
   - **Project URL** → вида `https://xxxx.supabase.co`
   - **anon public** ключ (длинная строка) — он **публичный**, для браузера это нормально.

> Сервисный (`service_role`) ключ **никуда не вставляйте** — он секретный.

## Шаг 3. Залить код на GitHub

1. Аккаунт на **github.com** → **New repository** → имя, например `aravt-tasks` → **Create**.
2. На странице репозитория: **Add file → Upload files** → перетащите **всё содержимое папки
   `aravt-supabase/`** (включая папку `src`). → **Commit changes**.

> Файл `.env` загружать не нужно — ключи мы зададим прямо в Vercel (шаг 4).

## Шаг 4. Подключить GitHub к Vercel и добавить переменные

1. Аккаунт на **vercel.com** (войдите через GitHub).
2. **Add New… → Project** → выберите репозиторий `aravt-tasks` → **Import**.
3. Vercel сам определит Vite (Framework Preset: **Vite**). Ничего менять не нужно.
4. Разверните **Environment Variables** и добавьте две переменные:

   | Name | Value |
   |------|-------|
   | `VITE_SUPABASE_URL` | ваш Project URL из шага 2 |
   | `VITE_SUPABASE_ANON_KEY` | ваш anon public ключ из шага 2 |

5. Нажмите **Deploy**. Через минуту получите ссылку вида `https://aravt-tasks.vercel.app`.

> Если позже поменяете переменные — зайдите в **Deployments → ⋯ → Redeploy**.

## Шаг 5. Подключить Supabase MCP в Claude.ai

1. На телефоне или в браузере откройте **Claude.ai → Settings → Connectors / Integrations**.
2. Найдите **Supabase** (или «Add custom connector» с MCP Supabase) и нажмите **Connect**.
3. Авторизуйтесь в Supabase и **выберите тот же проект**, где создали таблицу `tasks`.
4. Разрешите доступ. Теперь Claude может читать и писать в вашу базу.

> Точные названия пунктов в Claude.ai могут немного отличаться — ищите раздел про
> подключения/коннекторы и выбирайте Supabase.

## Шаг 6. Управлять задачами голосом

Откройте чат с Claude (можно надиктовывать голосом на iPhone) и говорите обычными словами —
Claude сам выполнит нужный SQL/запрос через MCP. Через секунду задача появится/изменится на сайте.

**Добавить задачу:**
> «Добавь в таблицу tasks задачу: КГМ-трейд, отгрузка серы 2000 тонн, категория work,
> приоритет high, статус inprogress, теги трейдинг и сера.»

Короче тоже работает — Claude дозаполнит по смыслу:
> «Запиши новую рабочую задачу: согласовать договор EcoSave, высокий приоритет.»

**Изменить статус:**
> «В таблице tasks поставь задаче EcoSave статус done.»

**Поменять приоритет:**
> «Измени приоритет задачи "Дубай — Анализ открытия компании" на medium.»

**Удалить задачу:**
> «Удали из tasks задачу про флиппинг домов.»

**Спросить состояние:**
> «Покажи все задачи в работе с высоким приоритетом.»

Допустимые значения, чтобы Claude попадал точно:
- **status:** `pending` (в очереди) · `inprogress` (в работе) · `done` (готово)
- **priority:** `high` · `medium` · `low`
- **category:** `work` (Работа) · `personal` (Личное)

## Локальный запуск (по желанию, не обязательно)

```bash
cd aravt-supabase
npm install
cp .env.example .env      # впишите свои URL и anon-ключ
npm run dev               # откроется http://localhost:5173
```

## Частые вопросы

**Сайт пишет «Нет связи с базой».** Проверьте, что в Vercel заданы обе переменные
`VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`, и сделайте **Redeploy**.

**Задача не появляется мгновенно.** Realtime включается в `supabase_setup.sql`
(строки `replica identity full` и `add table ... supabase_realtime`). Если запускали скрипт
частями — выполните эти две строки ещё раз.

**Кто-то может изменить мои задачи?** Сайт по anon-ключу умеет только читать и менять статус.
Добавление и удаление — через ваш подключённый Claude MCP. Ссылку на сайт никому лишнему не
давайте; при желании позже добавим вход по паролю.

**Хочу другие цвета/поля.** Палитра — в `tailwind.config.js`, разметка задачи — в `src/App.jsx`.
